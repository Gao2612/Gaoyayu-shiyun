# 个人诗云复现方案

## 一、项目定位

项目名称暂定为「个人诗云」。

它不是公共古诗数据库，也不做数万诗人的知识图谱。

它只承载一位作者的个人诗集，把每首诗变成三维宇宙中的一颗星。

核心体验是：

1. 进入页面，看到由个人全部诗作组成的星云。
2. 缩放、旋转、漫游，观察诗歌在时间和主题上的分布。
3. 点开任意星点，阅读完整诗歌及创作信息。
4. 查看诗歌之间基于主题、意象和时间形成的关系。
5. 可选开启摄像头，用手势控制星云。

一句话概括：

> 把一个人一生写过的诗，变成一片可以进入、漫游和阅读的宇宙。

## 二、参考视频拆解

### 1. 手势粒子视频

可复现的核心并非普通粒子动画，而是三层组合：

1. Three.js 或 WebGL 渲染大量发光粒子。
2. 摄像头识别手掌和手指关键点。
3. 手势参数实时驱动粒子的旋转、缩放和形态变化。

适合移植到个人诗云的交互：

- 手掌左右移动：旋转星云。
- 手掌上下移动：改变观察角度。
- 拇指和食指捏合：拉近或推远。
- 张开手掌：展开诗歌关系线。
- 握拳：收拢关系线并回到星云模式。
- 食指指向：移动选中光标。

### 2. 诗云视频

视频中的产品结构主要包括：

- 全屏三维星空。
- 每首诗对应一个可点击星点。
- 星点存在亮度、颜色和尺寸差异。
- 星点间可显示关系连线。
- 左右两侧提供筛选、统计和诗歌详情。
- 支持缩放、旋转、漫游和搜索定位。
- 点击对象后，镜头聚焦并展示文本信息。

个人版应保留三维漫游和诗歌详情。

公共诗人关系、朝代筛选和超大规模数据加载可以删除。

## 三、核心视觉模型

### 1. 星体定义

一首诗对应一颗星。

星体视觉属性建议如下：

| 视觉属性 | 数据来源 |
| --- | --- |
| 空间位置 | 诗歌文本的语义向量降维结果 |
| 颜色 | 主题或情绪分类 |
| 大小 | 字数、作者评分或阅读次数 |
| 亮度 | 喜爱程度或精选状态 |
| 光晕 | 是否包含重要意象 |
| 闪烁速度 | 创作时期或情绪强度 |

不要随机摆放诗歌。

空间应有可解释性：

- 意义相近的诗彼此靠近。
- 同一时期的诗形成局部轨迹。
- 主题变化会形成不同颜色的星团。

### 2. 推荐空间算法

首版不需要在线调用大模型。

在数据导入阶段离线完成：

1. 使用中文文本向量模型生成每首诗的语义向量。
2. 使用 UMAP 将向量降到三维。
3. 对三维坐标归一化。
4. 按主题和时间添加少量可控偏移。
5. 将最终坐标固化到数据文件或数据库。

推荐模型：

- `BAAI/bge-small-zh-v1.5`
- 或支持中文的云端 embedding API

少于 500 首诗时，也可以先用人工标签加确定性布局。

## 四、信息架构

### 1. 主场景

全屏显示个人诗歌星云。

默认状态保持安静，只有轻微漂浮、呼吸和星尘运动。

页面元素：

- 左上：作品名、作者名、诗歌总数。
- 顶部中央：搜索框。
- 左侧：年份、诗集、主题、情绪筛选。
- 右侧：当前选中诗歌详情。
- 右下：鼠标、触控、手势模式切换。
- 底部：时间轴。

### 2. 诗歌详情

点击星点后：

1. 镜头平滑飞向目标。
2. 目标星体变亮，附近星体变暗。
3. 右侧面板显示诗歌全文。
4. 关系线只连接最相关的 3 至 8 首诗。
5. 面板提供“前一首”“后一首”“回到星云”。

详情字段：

- 标题。
- 正文。
- 创作日期。
- 创作地点。
- 所属诗集。
- 主题标签。
- 情绪标签。
- 创作手记。
- 相关诗作。

### 3. 时间模式

将星体按创作日期拉成一条三维时间河流。

用户可沿时间轴回看作者不同阶段的作品。

这比公共诗云中的朝代筛选更适合个人诗集。

### 4. 星团模式

按主题聚类，例如：

- 爱情。
- 故乡。
- 城市。
- 四季。
- 离别。
- 自我。
- 梦境。

点击主题后，只保留对应星团，其余星体降低透明度。

## 五、手势交互方案

### 1. 技术选择

推荐使用：

- MediaPipe Tasks Vision。
- Hand Landmarker。
- 浏览器 `getUserMedia`。
- Three.js 相机控制器。

摄像头画面默认不上传服务器，只在本地浏览器处理。

### 2. 手势映射

| 手势 | 功能 |
| --- | --- |
| 单手移动 | 旋转视角 |
| 双指捏合 | 缩放 |
| 张开手掌 | 展开关系线 |
| 握拳 | 收起关系线 |
| 食指悬停 | 选择星点 |
| 食指停留一秒 | 打开诗歌 |
| 双手向外展开 | 展开整个星云 |
| 双手向内合拢 | 聚拢到当前诗集 |

### 3. 实现原则

手势只做增强，不作为唯一操作方式。

必须同时支持：

- 鼠标拖动。
- 滚轮缩放。
- 点击选择。
- 触屏手势。
- 键盘辅助操作。

首版先完成鼠标和触屏。

摄像头手势放在第二阶段，可以显著降低首版风险。

## 六、数据录入方案

### 1. 推荐录入格式

首版使用 Markdown 文件管理诗歌。

每首诗一个文件：

```md
---
id: poem-001
title: 夜航
writtenAt: 2024-11-03
location: 上海
collection: 未寄出的海
themes:
  - 城市
  - 夜晚
emotions:
  - 孤独
  - 平静
featured: true
rating: 5
---

这里是诗歌正文。
```

选择 Markdown 的原因：

- 便于人工校对和长期保存。
- 不绑定数据库。
- 支持 Git 版本管理。
- 后续可批量导入和导出。

### 2. 构建流程

构建脚本执行以下工作：

1. 读取全部 Markdown。
2. 校验字段和重复编号。
3. 提取纯文本。
4. 生成或读取语义向量。
5. 计算三维坐标。
6. 生成前端所需的压缩 JSON。
7. 建立标题、正文和标签搜索索引。

### 3. 数据结构

```ts
type Poem = {
  id: string;
  title: string;
  content: string;
  writtenAt: string;
  location: string | null;
  collection: string;
  themes: string[];
  emotions: string[];
  featured: boolean;
  rating: number;
  position: readonly [number, number, number];
  relatedPoemIds: string[];
};
```

## 七、技术架构

### 1. 前端

推荐：

- Next.js。
- TypeScript。
- React Three Fiber。
- Three.js。
- Drei。
- Zustand。
- Framer Motion。
- Fuse.js。
- Tailwind CSS。

React Three Fiber 负责三维场景。

普通 React 负责筛选器、详情面板和搜索界面。

### 2. 数据层

首版推荐纯静态数据：

- 诗歌保存在 `content/poems`。
- 构建后输出 `public/data/poems.json`。
- 网站可部署到任意静态托管服务。

个人诗集不需要先建设后台和数据库。

只有需要在线编辑、多设备同步或私密登录时，
再引入 PostgreSQL 和对象存储。

### 3. 渲染层

少于 2,000 首诗时：

- 使用 `InstancedMesh` 或自定义点精灵。
- 一次绘制全部星体。
- 使用 GPU shader 处理闪烁和呼吸。

关系线只渲染当前诗歌的局部关系。

不要一次渲染完整的全连接网络。

### 4. 页面模块

```text
app/
  page.tsx
components/
  galaxy/
    GalaxyCanvas.tsx
    PoemStars.tsx
    RelationLines.tsx
    CameraController.tsx
    GestureController.tsx
  panels/
    FilterPanel.tsx
    PoemDetailPanel.tsx
    SearchPanel.tsx
    Timeline.tsx
content/
  poems/
scripts/
  build-poem-data.ts
types/
  poem.ts
```

## 八、关键交互流程

### 1. 首次进入

1. 黑色背景中出现微弱星尘。
2. 星点从中心缓慢散开。
3. 显示个人诗集名称和一句引导语。
4. 用户操作后，引导文字淡出。

### 2. 搜索诗歌

1. 输入标题、正文片段或标签。
2. 搜索结果以列表展示。
3. 选择结果后，镜头飞向对应星体。
4. 自动打开诗歌详情。

### 3. 查看关系

1. 选中一首诗。
2. 计算好的相关诗作被点亮。
3. 曲线从当前星体延伸到相关星体。
4. 关系强度决定线条透明度。
5. 点击相关星体可继续跳转。

### 4. 切换时间模式

1. 星团平滑变形为时间河流。
2. 相机移动到适合观察的侧视角。
3. 底部显示年份刻度。
4. 拖动时间轴定位某个创作阶段。

## 九、视觉风格

不建议直接复制参考视频的金色古典星空。

个人版应形成自己的识别度。

推荐方向：

- 背景：墨黑、深蓝和极少量暖灰。
- 星体：按主题使用克制的低饱和颜色。
- 高亮：米白或金色。
- 文字：现代宋体搭配简洁无衬线字体。
- 面板：半透明墨色玻璃。
- 动效：缓慢、安静、有呼吸感。

诗歌阅读时应压低三维背景亮度，保证正文可读。

## 十、实施阶段

### 阶段一：可用原型

目标：

- 导入 20 至 50 首诗。
- 完成星空渲染。
- 完成鼠标旋转和缩放。
- 完成点击星体和阅读全文。
- 完成标题和标签搜索。

预计工作量：3 至 5 个开发日。

### 阶段二：完整个人诗云

目标：

- 完成全部诗歌导入。
- 加入语义坐标和主题星团。
- 加入时间模式。
- 加入相关诗作连线。
- 完成移动端适配。
- 完成加载和性能优化。

预计工作量：5 至 8 个开发日。

### 阶段三：手势与展示效果

目标：

- 接入摄像头。
- 完成基础手势识别。
- 加入粒子展开和聚拢效果。
- 加入首次进入动画。
- 增加演示和录屏模式。

预计工作量：3 至 5 个开发日。

## 十一、验收标准

### 功能

- 可导入个人全部诗歌。
- 可搜索标题、正文和标签。
- 可点击任意星体阅读诗歌。
- 可按诗集、主题、情绪和年份筛选。
- 可展示相关诗作。
- 可在桌面和移动端操作。
- 手势关闭时，全部功能仍可正常使用。

### 性能

- 首屏可交互时间控制在 3 秒左右。
- 桌面端目标帧率不低于 50 FPS。
- 中端手机目标帧率不低于 30 FPS。
- 切换模式时不重新请求全部数据。
- 不同时渲染无意义的大量关系线。

### 隐私

- 诗歌默认只来自项目内的数据源。
- 摄像头必须由用户主动开启。
- 摄像头画面不上传、不保存。
- 可选支持私有部署。

## 十二、最终建议

最合理的首版不是完全复制两个视频。

推荐组合为：

- 采用第二个视频的三维诗歌星系作为产品主体。
- 采用第一个视频的手势粒子作为增强交互。
- 数据只保留一位作者和个人诗集。
- 用时间、主题和意象代替诗人、朝代和人物关系。
- 先做可阅读、可搜索、可漫游的版本。
- 再加入摄像头手势和粒子形态变化。

首版最小范围建议：

1. 30 首真实诗歌。
2. 一个三维星云主场景。
3. 一个诗歌详情面板。
4. 搜索、主题筛选和时间筛选。
5. 鼠标、触屏两套操作。
6. 预留手势控制接口，但暂不强制接入。

这个范围已经能完整表达“个人诗歌宇宙”的核心价值，
同时避免首版陷入摄像头兼容、手势误触和过度特效。
