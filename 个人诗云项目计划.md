# 个人诗云项目计划

## 一、计划基线

项目目标：

> 建设一个只收录个人诗集的三维诗歌宇宙。

计划周期为一个月，共四周、二十个工作日。

默认配置：

- 一名全栈开发者。
- 首批录入 30 至 100 首诗。
- 优先完成桌面端，同时兼容移动端。
- 鼠标和触屏交互属于首版范围。
- 摄像头手势属于增强范围。
- 诗歌使用 Markdown 文件录入。
- 首版采用静态数据，不建设管理后台。

## 二、一个月总体计划

| 周期 | 阶段目标 | 核心交付物 |
| --- | --- | --- |
| 第一周 | 建立项目和基础星云 | 可漫游的三维诗歌原型 |
| 第二周 | 完成核心产品功能 | 可搜索、筛选和阅读的个人诗云 |
| 第三周 | 完成数据关系和体验优化 | 语义星团、时间模式、移动端 |
| 第四周 | 加入手势并完成发布 | 手势体验、性能优化、正式版本 |

### 月度验收目标

- 全部个人诗歌能够正常导入。
- 每首诗在三维空间中具有稳定坐标。
- 可通过点击、搜索和筛选定位诗歌。
- 可阅读完整正文和创作信息。
- 可查看主题星团、时间轨迹和相关诗作。
- 桌面端和移动端均可正常操作。
- 摄像头手势可以独立开启和关闭。
- 摄像头数据不上传服务器。
- 项目完成生产构建并可公开或私有部署。

## 三、第一周计划

### 周目标

建立完整工程骨架，完成三维星空、数据读取和基础交互。

### 第一天：需求冻结与工程初始化

工作内容：

- 确认首批诗歌数量和字段。
- 确认项目名称、作者名和视觉基调。
- 初始化 Next.js 和 TypeScript。
- 安装 React Three Fiber、Drei 和 Zustand。
- 建立目录结构和基础页面。
- 配置代码检查和生产构建命令。

交付物：

- 可启动的项目。
- 三维画布占位页面。
- 诗歌数据类型定义。
- 项目目录结构。

验收标准：

- 本地开发服务器可以正常启动。
- 页面无运行时错误。
- TypeScript 检查和生产构建通过。

### 第二天：诗歌录入与构建流程

工作内容：

- 定义 Markdown 元数据格式。
- 创建 5 至 10 首示例诗歌。
- 编写诗歌读取和字段校验脚本。
- 检查空标题、无正文和重复编号。
- 输出前端需要的 JSON 数据。

交付物：

- `content/poems` 诗歌目录。
- 诗歌数据构建脚本。
- 前端诗歌数据文件。

验收标准：

- Markdown 能稳定转换为结构化数据。
- 错误数据能够给出明确错误信息。
- 构建结果包含标题、正文、日期和标签。

### 第三天：基础三维星空

工作内容：

- 创建星空背景和环境星尘。
- 将每首诗渲染为一个星点。
- 使用固定种子生成临时坐标。
- 加入星点大小、颜色和亮度差异。
- 使用实例化渲染减少绘制次数。

交付物：

- 可显示全部示例诗歌的三维星空。
- 星点视觉属性映射逻辑。

验收标准：

- 每首诗只对应一个星点。
- 刷新页面后星点位置保持不变。
- 桌面端运行流畅，无明显卡顿。

### 第四天：相机和选中交互

工作内容：

- 完成鼠标拖动旋转。
- 完成滚轮缩放。
- 限制最小和最大观察距离。
- 使用射线检测识别点击的星点。
- 加入悬停高亮和选中状态。
- 加入镜头聚焦动画。

交付物：

- 可旋转、缩放和点击的星云。
- 星体悬停和选中反馈。

验收标准：

- 用户能稳定点击目标星体。
- 镜头不会穿过星云或无限远离。
- 选中星体后有清晰视觉反馈。

### 第五天：诗歌详情和第一周验收

工作内容：

- 创建诗歌详情侧栏。
- 展示标题、正文、日期和标签。
- 加入关闭详情和回到星云功能。
- 处理长诗滚动和空字段。
- 完成第一周集成检查。

交付物：

- 可阅读诗歌全文的三维原型。
- 第一周演示版本。

验收标准：

- 点击任意星点能够打开正确诗歌。
- 长诗不会破坏页面布局。
- 关闭详情后可以继续漫游。

## 四、第二周计划

### 周目标

完成搜索、筛选、诗集导航和核心阅读流程。

### 第六天：全文搜索

工作内容：

- 接入 Fuse.js。
- 建立标题、正文和标签索引。
- 创建搜索框和结果列表。
- 选择结果后定位对应星体。
- 自动打开诗歌详情。

交付物：

- 诗歌全文搜索功能。

验收标准：

- 标题、正文片段和标签均可检索。
- 搜索结果可以正确定位星体。
- 无结果时提供明确提示。

### 第七天：多维筛选

工作内容：

- 添加诗集筛选。
- 添加主题筛选。
- 添加情绪筛选。
- 添加年份筛选。
- 未命中的星体降低透明度。
- 提供清除全部筛选功能。

交付物：

- 左侧筛选面板。
- 星云筛选状态。

验收标准：

- 多个筛选条件能够组合使用。
- 清除筛选后恢复全部诗歌。
- 筛选不会修改原始诗歌数据。

### 第八天：诗集导航

工作内容：

- 建立诗集列表。
- 显示各诗集的诗歌数量。
- 点击诗集后聚焦对应星团。
- 在详情中加入前一首和后一首。
- 加入随机漫游入口。

交付物：

- 诗集导航功能。
- 连续阅读功能。

验收标准：

- 诗集数量统计准确。
- 前后切换顺序稳定。
- 随机漫游不会选择隐藏诗歌。

### 第九天：视觉系统

工作内容：

- 确定背景、星体和高亮颜色。
- 设计详情面板和筛选面板。
- 配置中文字体。
- 加入加载动画和空状态。
- 降低阅读时的三维背景亮度。
- 统一按钮、间距和动效时间。

交付物：

- 第一版完整视觉系统。
- 可稳定阅读的界面。

验收标准：

- 正文具有足够对比度。
- 页面不存在难以识别的控件。
- 动效不会妨碍阅读和操作。

### 第十天：第二周集成验收

工作内容：

- 录入至少 30 首真实诗歌。
- 检查搜索和筛选结果。
- 检查诗歌详情内容。
- 修复数据、交互和布局问题。
- 完成桌面端核心流程演示。

交付物：

- 可实际使用的个人诗云首版。

验收标准：

- 从进入页面到阅读诗歌的流程完整。
- 所有真实诗歌均能被检索和打开。
- 核心操作没有阻断性错误。

## 五、第三周计划

### 周目标

建立可解释的诗歌空间，加入时间模式、关系线和移动端。

### 第十一天：语义向量预处理

工作内容：

- 清洗诗歌正文。
- 生成每首诗的中文语义向量。
- 缓存向量结果。
- 记录模型版本和处理时间。
- 为短诗和空标签设置明确规则。

交付物：

- 诗歌语义向量数据。
- 可重复执行的预处理脚本。

验收标准：

- 每首有效诗歌都有向量。
- 重复执行不会无条件重新请求模型。
- 失败时能够定位具体诗歌。

### 第十二天：三维语义布局

工作内容：

- 使用 UMAP 将向量降到三维。
- 对坐标执行归一化。
- 处理重叠星点。
- 添加主题和时间的轻量偏移。
- 固化最终三维坐标。

交付物：

- 具有语义意义的星云布局。

验收标准：

- 相似诗歌整体上彼此接近。
- 星点不存在大面积完全重叠。
- 刷新和重新构建后布局保持稳定。

### 第十三天：相关诗作关系线

工作内容：

- 计算诗歌相似度。
- 为每首诗生成 3 至 8 个相关诗歌。
- 选中诗歌时显示局部关系线。
- 用透明度表达关系强度。
- 点击相关星体可以继续跳转。

交付物：

- 诗歌关系网络。

验收标准：

- 不渲染完整全连接网络。
- 关系线与目标星体准确对应。
- 切换诗歌后旧关系线被正确清理。

### 第十四天：时间模式

工作内容：

- 按创作日期生成时间布局。
- 创建底部时间轴。
- 加入星云模式和时间模式切换。
- 实现星体位置平滑过渡。
- 支持按年份快速定位。

交付物：

- 三维创作时间河流。

验收标准：

- 日期有效的诗歌顺序正确。
- 无明确日期的诗歌有独立区域。
- 模式切换不会丢失当前筛选状态。

### 第十五天：移动端适配

工作内容：

- 完成触屏旋转和双指缩放。
- 将左右侧栏改为抽屉或底部面板。
- 调整移动端字体和点击区域。
- 限制移动端粒子数量和像素比。
- 检查横屏和竖屏。

交付物：

- 移动端可用版本。
- 第三周演示版本。

验收标准：

- 手机可以完成搜索、筛选和阅读。
- 触控操作不会与页面滚动冲突。
- 中端手机目标帧率不低于 30 FPS。

## 六、第四周计划

### 周目标

接入手势控制，完成性能、隐私、测试和发布。

### 第十六天：摄像头和手部识别

工作内容：

- 接入 MediaPipe Hand Landmarker。
- 建立摄像头开启和关闭按钮。
- 处理授权拒绝和设备不可用。
- 在本地输出手部关键点。
- 明确展示摄像头隐私说明。

交付物：

- 可独立启停的手部识别模块。

验收标准：

- 不主动请求摄像头权限。
- 关闭后摄像头设备被正确释放。
- 摄像头画面和关键点不上传服务器。

### 第十七天：手势控制

工作内容：

- 单手移动映射到视角旋转。
- 双指捏合映射到缩放。
- 张开手掌展开关系线。
- 握拳收起关系线。
- 加入平滑、阈值和防抖。
- 提供鼠标和手势模式切换。

交付物：

- 基础手势漫游功能。

验收标准：

- 手势抖动不会导致镜头剧烈跳动。
- 手势关闭后鼠标功能完整可用。
- 无法识别手部时不会触发随机操作。

### 第十八天：展示动效和演示模式

工作内容：

- 加入首次进入的星云展开动画。
- 加入诗歌聚焦和离场动画。
- 加入手势触发的粒子聚散效果。
- 创建适合录屏的自动漫游模式。
- 提供关闭高强度动效的选项。

交付物：

- 可用于展示和录屏的视觉版本。

验收标准：

- 动画结束后状态准确。
- 自动漫游可以随时退出。
- 低性能设备可以关闭增强动效。

### 第十九天：性能和质量检查

工作内容：

- 检查加载体积。
- 优化实例化星体和材质。
- 限制设备像素比。
- 延迟加载摄像头模块。
- 检查桌面端和移动端核心流程。
- 检查错误提示、空状态和边界数据。

交付物：

- 性能优化版本。
- 验收问题清单及修复结果。

验收标准：

- 首屏可交互时间控制在约 3 秒。
- 桌面端目标帧率不低于 50 FPS。
- 页面无阻断性控制台错误。
- 生产构建成功。

### 第二十天：发布和月度验收

工作内容：

- 完成全部诗歌最终导入。
- 执行生产构建。
- 部署公开版或私有版。
- 检查线上资源、字体和摄像头权限。
- 完成数据备份和部署说明。
- 按月度验收目标逐项检查。

交付物：

- 正式上线版本。
- 完整个人诗歌数据。
- 项目运行和部署说明。

验收标准：

- 线上地址可以正常访问。
- 诗歌数据、搜索和筛选结果正确。
- HTTPS 环境下摄像头功能可用。
- 关键功能在主流浏览器中通过检查。

## 七、每日工作节奏

建议每个工作日按以下节奏执行：

### 上午

- 检查前一日遗留问题。
- 明确当天唯一核心交付物。
- 完成数据结构和主要功能。

### 下午

- 完成交互、样式和边界情况。
- 使用真实诗歌验证功能。
- 修复当天发现的问题。

### 当日结束前

- 运行类型检查。
- 运行生产构建。
- 检查核心用户流程。
- 更新任务状态和风险记录。

每日完成标准：

- 功能可以被真实操作验证。
- 不保留已知的阻断性错误。
- 当天变更不破坏此前完成的流程。

## 八、每周验收节点

### 第一周验收

- 能看见由诗歌组成的三维星空。
- 能旋转、缩放和点击星点。
- 能阅读完整诗歌。

### 第二周验收

- 能搜索和筛选诗歌。
- 能按诗集连续阅读。
- 至少录入 30 首真实诗歌。

### 第三周验收

- 星体位置具有语义或主题意义。
- 能查看相关诗歌和创作时间轨迹。
- 移动端可完成核心操作。

### 第四周验收

- 手势控制可独立启停。
- 性能和隐私要求达标。
- 正式版本完成部署。

## 九、优先级与范围控制

### 必须完成

- 诗歌录入和校验。
- 三维星云。
- 鼠标和触屏漫游。
- 星体点击和全文阅读。
- 搜索和筛选。
- 移动端适配。
- 生产部署。

### 应当完成

- 语义星团。
- 相关诗作。
- 时间模式。
- 首次进入动画。

### 可以延后

- 摄像头手势。
- 自动漫游和录屏模式。
- 在线编辑后台。
- 用户登录。
- 阅读统计。
- 社交分享。

如果进度出现偏差，首先延后摄像头手势和展示动效。

不得压缩诗歌阅读、搜索、筛选和移动端适配的时间。

## 十、主要风险

| 风险 | 影响 | 处理方式 |
| --- | --- | --- |
| 诗歌元数据不完整 | 时间和主题模式失真 | 首周完成字段校验 |
| 星体点击困难 | 无法正常阅读 | 扩大不可见命中区域 |
| 语义布局不稳定 | 每次构建位置变化 | 固定随机种子并保存坐标 |
| 粒子数量过多 | 手机帧率下降 | 实例化渲染和分级质量 |
| 手势误触 | 镜头操作失控 | 阈值、防抖和明确模式开关 |
| 摄像头权限被拒绝 | 手势不可用 | 保留完整鼠标和触屏功能 |
| 中文字体加载过慢 | 首屏延迟 | 字体子集化和延迟加载 |
| 首批诗歌整理延迟 | 功能无法真实验收 | 第一周先准备 30 首诗 |

### 里程碑三：第 15 个工作日

完成语义星团、关系线、时间模式和移动端。
